# Workflows

Workflows are simple declarative sequences in `config/apps.json`.

They are intentionally small:

- No loops.
- No conditions.
- No JavaScript eval.
- No scripting engine.
- No hidden state beyond inherited `app` and `deviceId`.

## Structure

```json
{
  "workflows": {
    "appSmoke": [
      {
        "tool": "android_launch_app",
        "args": {}
      },
      {
        "tool": "android_capture_state",
        "args": {}
      }
    ]
  }
}
```

Run:

```json
{ "app": "myapp", "workflow": "appSmoke", "deviceId": "SERIAL" }
```

## Context Inheritance

When a workflow runs:

- `app` is inherited by every step.
- `deviceId` is inherited by every step.
- A step can override inherited values by setting them in `args`.

## Reports

Each run creates:

```text
workflow-reports/TIMESTAMP-app-workflow/
```

Contents include:

- `workflow.json`
- `execution-log.json`
- `metadata.json`
- captures/reports generated by steps

## Generic Workflow Examples

### appSmoke

```json
[
  { "tool": "android_launch_app", "args": {} },
  { "tool": "android_capture_state", "args": {} },
  { "tool": "android_generate_report", "args": { "lines": 300 } }
]
```

### launchAndCapture

```json
[
  { "tool": "android_launch_app", "args": {} },
  { "tool": "android_capture_state", "args": {} }
]
```

### uiSnapshot

```json
[
  { "tool": "android_screenshot", "args": {} },
  { "tool": "android_ui_dump", "args": {} }
]
```

### logcatSnapshot

```json
[
  { "tool": "android_clear_logcat", "args": {} },
  { "tool": "android_launch_app", "args": {} },
  { "tool": "android_read_logcat", "args": { "lines": 300 } }
]
```

### debugCapture

```json
[
  { "tool": "android_launch_app", "args": {} },
  { "tool": "android_send_debug_intent", "args": { "intent": "dumpState" } },
  { "tool": "android_generate_report", "args": { "lines": 500 } }
]
```

## Best Practices

- Prefer `android_wait_for_ui` over fixed sleeps.
- Prefer `android_tap_resource` over coordinate taps when resource ids are available.
- Use debug intents for stable app-specific actions.
- Generate reports at the end of smoke workflows.
- Keep workflows short and explicit.
